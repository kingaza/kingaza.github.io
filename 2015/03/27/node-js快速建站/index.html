现在node.js很火很火，连微软的Visual Studio都对它提供开发环境。如果使用node.js，那么以前对web的概念，都还是我们熟悉的一些关键词:http, request, response, html, template engine, url routing, MVC, GET, POST, Database……唯一不同的是, 这里只使用javascript就可以驱动着以上的一切.

Express是目前最流行的基于Node.js的Web开发框架, 使用它可以快速地搭建网站原型，它是一个node.js模块，采用npm全局模块进行安装。而且他十分精简，在4.x版本下周有2、3百行的代码。
再利用bootstrap美化界面，jquery简化编程，还有利用mongodb来做持久化……这个组合想想都很强大~~

目录结构

接下来，我们详细看一下express4项目的结构、配置和使用。

bin, 存放启动项目的脚本文件
node_modules, 存放所有的项目依赖库。
public，静态文件(css,js,img)
routes，路由文件(MVC中的C,controller)
views，页面文件(Ejs模板)
package.json，项目依赖配置及开发者信息
app.js，应用核心配置文件

package.json项目配置

package.json用于项目依赖配置及开发者信息，scripts属性是用于定义操作命令的，可以非常方便的增加启动命令，比如默认的start，用npm start代表执行node ./bin/www命令。

查看package.json文件。

查看app.js文件，我们也能看到不少变化。在app.js中，原来调用connect库的部分都被其他的库所代替，例如serve-favicon、morgan、cookie-parser、body-parser。此外项目启动代码也被移到./bin/www的文件，www文件也是一个node的脚本，用于分离配置和启动程序。
