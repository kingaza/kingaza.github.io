树莓派：编程语言

By John Doe

發表於 2014-09-18

文章目錄

准确来说，树莓派并没有标准的开发语言。它本质上是一台安装Linux系统的电脑，虽然CPU是ARM的。那么理论上普通Linux机器上能用的语言，树莓派上也可以使用。

  1. Python
    所谓的胶水语言，因为它能够把各种语言胶合起来，使用方便，因此哪儿都用得上。用Python处理一些简短程序类似与操作系统的交互编程工作最省时省力。
    先看一下版本

    pi@raspberrypi ~ $ python -V
Python 2.7.3

    好吧，试试print^_^

    print 'hello world'
hello world

    Python是跨平台的，在树莓派上运行各种语言特性，跟在Windows下丝毫没发现区别。如果在树莓派上开发小工具的话，数据库跟网络往往是必须的。
    对于很多小项目，并不需要开发复杂的关系型数据库。因此python的dbm、Pickle、Shelve等很能派上用场。anydbm将数据文件与一个“dict-like”对象关联起来，操作这个“dict-like”对象，就像操作dict对象一样，最后可以将“dict-like”的数据持久化到文件。对这个”dict-like”对象进行操作的时候，key和value的类型必须是字符串。

    >>> import anydbm
>>> file = anydbm.open('name','c')
>>> file['zhang'] = 'san'
>>> file.keys()
['zhang']
>>> file['zhang']
'san'

    Pickle模块实现了基本的数据序列和反序列化，Shelve则是anydbm+Pickle的增强版，它支持在”dict-like”对象中存储任何可以被Pickle序列化的对象，但key也必须是字符串。

    import shelve

def CreateDB():
    try:
        db = shelve.open('db.dat', 'c')
        db['int'] = 1
        db['float'] = 2.3
        db['string'] = "I am Shelve."
        db['key'] = 'value'
    finally:
        db.close()

def LoadDB():
    db = shelve.open('db.dat', 'r')
    for item in db.items():
        print item
    db.close()

if __name__ == '__main__':
    CreateDB()
    LoadDB()

    至于网络访问，则是建议使用requests库实现get/post请求。这是一个简单好用的网络通信工具。

    import requests
r = requests.get(apiurl,headers=apiheaders)

  2. C/C++
    既然说到胶水，那么必须得有要连接起来的基础件。这就到了C/C++大显身手的时候了。偶然看到openFrameworks，我和我的小伙伴们表示，看完这个视频都被惊呆了。

它就是用openFrameworks开发的，一个以用户交互见长，是一个拥有艺术家加黑客气质的框架。
从0.80开始，openFrameworks支持Raspberry Pi。偶然会觉得他们有相同的气质，大概也是一对好基友。
然后就是一时冲动，在树莓派上安装了Code::Blocks，居然也能跑起来了。但强烈怀疑编译运行速度。。。
