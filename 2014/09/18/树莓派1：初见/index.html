树莓派1：初见

  • Raspberry Pi
  • 智能硬件

Raspberry Pi(中文名为“树莓派”,简写为RPi，或者RasPi/RPi)是为学生计算机编程教育而设计，只有信用卡大小的卡片式电脑，其系统基于Linux。它是由英国的树莓派基金会（Raspberry Pi Foundation ）开发，意在提升学校在CS（Computer Science）上的教学。

1.前言

最早是在csdn上的一篇文章《三大主流开源硬件对比：Arduino vs BeagleBone vs Raspberry Pi》， 知道了这个神奇的东西，惊喜的发现一个这么小的东西居然可以提供如此强大的功能，它自带CPU、GPU、网络接口，可谓外表柔弱，内“心”强大。碰巧跟同学聊天，得知他打算用Arduino做运动导轨，顿时很受启发，好歹俺也是电子出身的，这是老本行，亲切熟悉得很。而且近期项目中软硬件联调频繁，I2C、SPI什么的多少了解了一点。项目进展不顺利，百无聊赖之际就想涉猎点别的东西。树莓派玩的就是软件，玩的是社区而不是硬件，刚好适合俺们这种悲催的程序猿。

2.安装

首先要有树莓派，还要若干温度湿度摄像头等传感器。树莓派由于BCM2835的限制，仅有HDMI和复合视频两种输出形式。可以通过HDMI直接接到电视等显示设备上，也可以通过ssh或者 vnc在局域网内部访问。俺就选用了ssh，简单省事，不需要折腾各种线缆。值得注意的是，千万不要为树莓派配备不接电源的HDMI转VGA转换器。eLinux上的一篇wiki警告说：不带电源的HDMI→VGA转换器可能损坏树莓派。这是因为HDMI连接器的第18针是一个+5V的供电，但是根据规范，它最多提供50mA的电流。而树莓派对第18针的供电，是将+5V输入，串接一个二极管D1(BAT54)作为保护送往18针。BAT54最高允许200mA电流（够宽松了其实）。但是市售不带电源的HDMI→VGA转换器，往往会从HDMI的18针吸取高至500mA的电流，这样就会远远超出D1的耐受能力，而最终烧毁D1。
Raspberry Pi没有自带硬盘或者闪存，其操作系统必须安装到自行购买的SD卡上。这是为了控制成本，但这做法很值得商榷，因为实际上给用户带来了不方便。值得注意的有些SD卡会有兼容问题，购买前最好先参考《RPi SD cards》。操作系统以Linux为主，作为菜鸟还是选择一个官方量身定制的Raspian “wheezy”，这是一个在Debian基础上优化的版本，安装过程相当傻瓜式，几乎是一路Next到Finish。

3.开发语言

准确来说，树莓派并没有标准的开发语言。它本质上是一台安装Linux系统的电脑，虽然CPU是ARM的。那么理论上普通Linux机器上能用的语言，树莓派上也可以使用。
Python有胶水语言的称呼，因为它能够把各种语言胶合起来，使用方便，因此哪儿都用得上。用Python处理一些简短程序类似与操作系统的交互编程工作最省时省力。因此，在树莓派上python十分流行。
Python是跨平台的，在树莓派上运行各种语言特性，跟在Windows下丝毫没发现区别。如果在树莓派上开发小工具的话，数据库跟网络往往是必须的。
对于很多小项目，并不需要开发复杂的关系型数据库。因此python的dbm、Pickle、Shelve等很能派上用场。anydbm将数据文件与一个“dict-like”对象关联起来，操作这个“dict-like”对象，就像操作dict对象一样，最后可以将“dict-like”的数据持久化到文件。对这个”dict-like”对象进行操作的时候，key和value的类型必须是字符串。

Pickle模块实现了基本的数据序列和反序列化，Shelve则是anydbm+Pickle的增强版，它支持在”dict-like”对象中存储任何可以被Pickle序列化的对象，但key也必须是字符串。

import shelve

def CreateDB():
    try:
        db = shelve.open('db.dat', 'c')
        db['int'] = 1
        db['float'] = 2.3
        db['string'] = "I am Shelve."
        db['key'] = 'value'
    finally:
        db.close()

def LoadDB():
    db = shelve.open('db.dat', 'r')
    for item in db.items():
        print item
    db.close()

if __name__ == '__main__':
    CreateDB()
    LoadDB()

至于网络访问，则是建议使用requests库实现get/post请求。这是一个简单好用的网络通信工具。

4.硬件接口

玩树莓派，基本上也是玩各种外设，典型的如温度湿度传感器、继电器、步进电机等等。这些外设就可以通过GPIO接口提供控制信号或者传输数据。GPIO，英文全称为General-Purpose IO ports，也就是通用IO口。在嵌入式系统中常常有数量众多，但是结构比较简单的外部设备/电路，对这些设备/电路有的需要CPU为之提供控制手段， 有的则需要被CPU用作输入信号。而且，许多这样的设备/电路只要求一位，即只要有开/关两种状态就够了，比如灯亮与灭。对这些设备/电路的控制，使用传统的串行口或并行口都不合适。所以在微控制器芯片上一般都会提供一个“通用可编程IO接口”。GPIO接口至少有两个寄存器，“通用IO控制寄存器”与“通用IO数据寄存器”。
幸运的事，大牛们已经提供了相当多的工具。这就是开源的威力：

  • PRi.GPIO
  • WebIOPi
  • wiringPi
